# Confidence and Citations

Use this file when packaging the final answer.

## Citation Rules

### Full reports and decision memos
- cite major claims inline
- include a source list or bibliography when helpful
- prefer direct links to the strongest upstream source

### Chat-native scout outputs
- put URLs inline next to the item
- do not append a separate "Sources" section unless explicitly asked
- keep each item self-contained

### All outputs
- if a key claim has only one supporting source, label it as fragile or lightly supported
- if sources conflict, show the conflict instead of flattening it
- separate what the source says from your own synthesis

## Confidence Rules

Map each major conclusion to one of:

```text
High confidence
Medium confidence
Low confidence
Speculative
```

Do not overstate.

## Distinction Rules

```text
Fact        = directly supported by source evidence
Synthesis   = pattern derived from multiple sources
Inference   = reasoned conclusion extending beyond explicit source wording
Speculation = scenario, emerging signal, or weakly supported projection
```

Make these boundaries clear whenever the answer could influence a real decision.

## Formatting Rules

- Use visual hierarchy with clear section headers.
- Use emoji bullets sparingly but helpfully.
- Wrap tables in triple-backtick code blocks.
- For Discord/Telegram-friendly outputs, prefer short sections and inline URLs.
- For long research, lead with the answer, not the process.
